JARVIS (ADA) TECHNICAL SPECIFICATION: AGENTIC AI & SUPERVISED INTENT CLASSIFICATION
================================================================================

1. PROJECT IDENTITY
-------------------
Jarvis (ADA) is a local-first, agentic AI assistant designed to solve the critical challenges of cloud latency and connectivity dependency. By utilizing a unique "Dual-Brain" architecture, the system balances high-reasoning cloud capabilities with ultra-low-latency local execution. ADA serves as an intelligent intermediary between the user's natural language and the operating system's low-level APIs.

2. SUPERVISED LEARNING PIPELINE
-------------------------------
### Preprocessing
The pipeline begins with real-time data acquisition from audio sensors.
- **Voice/Text Cleaning:** Audio streams are normalized using NumPy and processed through noise-reduction filters. Textual inputs are tokenized and stripped of filler words (stop-word removal) to isolate the core command.
- **Feature Engineering:** The system captures real-time "system state variables" (CPU Load, RAM Usage, Battery Status) and transforms them into feature vectors. These features help the model decide whether a task is "Heavy" (Cloud) or "Utility" (Local).

### EDA & Visualization
Exploratory Data Analysis (EDA) is performed to understand system behavior and model performance.
- **Distribution Plots:** Used to analyze the frequency of specific command classes (e.g., "Open App" vs. "System Query") to optimize cache hits for the Local Core.
- **Bivariate Charts:** Visualization of "Task Complexity vs. Latency" to demonstrate the performance gain achieved by bypassing cloud APIs for local tasks.

### Model Building
The "Intent Router" functions as a supervised multi-class classifier. While the system utilizes LLMs for inference, the underlying logic maps to established supervised models:
- **Random Forest:** Used for multi-branch decision trees to route commands to the correct OS API.
- **SVM & KNN:** Employed conceptually for string-distance matching and intent clustering to map natural language to structured JSON labels (Classification).

3. THE DUAL-BRAIN DECISION LOGIC
--------------------------------
The system employs a probabilistic routing mechanism to ensure maximum efficiency.
- **Intent Probability Threshold:** For every user request, the Intent Router calculates a confidence score ($P$). If $P > 0.8$ for a known local utility task, the system triggers the **Local Core (Ollama/Llama 3.2)**, avoiding internet round-trips.
- **Routing Process:**
    - **Local Path:** Intent Router -> Local Core -> OS Libraries (PyAutoGUI, PSUtil, Subprocess).
    - **Cloud Path:** Intent Router (Low Confidence) -> Gemini API -> Reasoning -> Execution Engine.

4. MODEL EVALUATION & COMPARISON
--------------------------------
The performance of the classification models is tracked using a standard evaluation matrix.

*Note: A Confusion Matrix is utilized to identify edge cases where "General Chat" intents are misclassified as "System Commands".*

5. TECHNICAL JUSTIFICATION: CLOUD VS. LOCAL
-------------------------------------------
This project is fundamentally incapable of running on cloud-only environments like **Google Colab**. The mechanical necessity of the system requires:
- **Direct Hardware Access:** Low-level interface with audio drivers (SAPI5/PyAudio) which are virtualized or unavailable in headless cloud containers.
- **OS-Level Drivers:** The use of `PyAutoGUI` for mouse/keyboard emulation requires a local display buffer and desktop environment permissions.
- **Hardware Proximity:** Real-time monitoring of local CPU/Battery sensors via `PSUtil` is only possible on the physical host machine.

6. CONCLUSION
-------------
Jarvis (ADA) demonstrates the real-world application of automating daily computing tasks through agentic AI. By treating intent recognition as a supervised classification problem and implementing a dual-brain routing strategy, the project achieves a balance between intelligence, speed, and privacy. This architecture serves as a blueprint for the next generation of local-first intelligent assistants.
